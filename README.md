# LLM-Powered Prompt Router for Intent Classification

A Python service that intelligently routes user requests to specialized AI personas. The system uses a two-step process: **Classify** (detect intent via a lightweight LLM call), then **Respond** (route to an expert persona and generate a focused response).

## Design Overview

### Architecture

```
User Message → classify_intent() → Intent + Confidence
                    ↓
            route_and_respond()
                    ↓
    ┌───────────────┼───────────────┐
    │               │               │
    ▼               ▼               ▼
 Code Expert   Data Analyst   Writing Coach   Career Advisor   Unclear → Clarify
    │               │               │               │
    └───────────────┴───────────────┴───────────────┘
                    ↓
            Final Response + route_log.jsonl
```

### Key Components

- **`classify_intent(message)`**: Calls a fast, cheap LLM (e.g., gpt-4o-mini) to return `{"intent": "code"|"data"|"writing"|"career"|"unclear", "confidence": 0.0-1.0}`. Handles malformed JSON by defaulting to `unclear`.
- **`route_and_respond(message, intent)`**: Selects the appropriate expert system prompt and makes a second LLM call to generate the final response. For `unclear`, generates a clarifying question instead.
- **Expert Prompts**: Four distinct personas (code, data, writing, career) defined in `prompts.py`, each with a sharp, opinionated role and output format.

### Supported Intents

| Intent   | Persona        | Use Case                                      |
|----------|-----------------|-----------------------------------------------|
| `code`   | Code Expert     | Programming, debugging, algorithms            |
| `data`   | Data Analyst    | Statistics, datasets, visualizations          |
| `writing`| Writing Coach   | Text feedback, clarity, structure             |
| `career` | Career Advisor  | Job search, interviews, career planning       |
| `unclear`| —               | Asks user to clarify their need               |

## Setup

### Prerequisites

- Python 3.9+
- OpenAI API key

### Installation

```bash
cd prompt-router
pip install -r requirements.txt
cp .env.example .env
# Edit .env and set OPENAI_API_KEY
```

### Environment Variables

| Variable              | Required | Default        | Description                          |
|-----------------------|----------|----------------|--------------------------------------|
| `OPENAI_API_KEY`      | Yes      | —              | OpenAI API key                       |
| `CLASSIFIER_MODEL`    | No       | `gpt-4o-mini`  | Model for intent classification      |
| `GENERATION_MODEL`    | No       | `gpt-4o`       | Model for response generation        |
| `ROUTE_LOG_PATH`      | No       | `route_log.jsonl` | Path to JSON Lines log file      |
| `CONFIDENCE_THRESHOLD`| No       | `0`            | If confidence < threshold, treat as unclear |

## Running the Application

### CLI (Interactive)

```bash
python app.py
```

Type messages and press Enter. Type `quit` to exit.

### API Server

```bash
uvicorn api:app --reload --host 0.0.0.0 --port 8000
```

- **POST /chat**: `{"message": "your message"}` → `{"intent": "...", "confidence": 0.9, "response": "..."}`
- **GET /health**: Health check

### Docker

```bash
# Build
docker build -t prompt-router .

# Run CLI interactively
docker run -it --env-file .env -v $(pwd)/route_log.jsonl:/app/route_log.jsonl prompt-router

# Run API
docker compose --profile cli up prompt-router      # CLI
docker compose up prompt-router-api               # API on port 8000
```

## Manual Override

You can bypass the classifier by prefixing your message with an intent:

- `@code Fix this bug: ...`
- `@data Analyze these numbers: ...`
- `@writing Review this paragraph: ...`
- `@career I need interview tips`

## Logging

Every request is appended to `route_log.jsonl` (or the path in `ROUTE_LOG_PATH`). Each line is a JSON object:

```json
{"intent": "code", "confidence": 0.95, "user_message": "how do i sort a list?", "final_response": "..."}
```

## Testing

Run the test suite (15+ messages) to generate `route_log.jsonl`:

```bash
python test_router.py
```

Ensure `OPENAI_API_KEY` is set in `.env` or the environment. A sample `route_log.jsonl` with example entries is included; running the tests will overwrite it with live results.
